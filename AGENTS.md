# AGENTS.md

> 规范版本：V1.7

## 1. 规范效力

本仓库实现 UAV-USV 异构港区水空巡检研究的冻结规范。

实质性结论按以下顺序生效：

1. 第一项：时间结构、空间范围、上下层职责。
2. 第二项：任务真实性、任务族、实证港口。
3. 第三项：空间粒度、上层调度任务单元。
4. 第四至第七项：能力、任务表示、依赖、截止时间与重访。
5. 第八项：时间、能量、返航与补能。
6. 第八至第十二项结构 V2：仅更新后续编号、审查顺序和跨项边界。
7. 第九项 V1.3：事件驱动上层调度、完整可行关系和硬约束处理。
8. `V1.4-A9`：引入学习型软锁定 Top-K 任务波次。
9. `V1.5-A9T`：曾将双时间尺度中央图 PPO 预设为最终上层方法，并将 MAPPO/HAPPO 限定为基线。
10. `V1.6-A9D`：曾加入反馈时延、信息年龄和任务响应时延。
11. `V1.7-A9C`：`REPLACES` 第九项中“中央 PPO 预设为最终方法、MAPPO 仅作基线”的条款；`REPLACES` V1.6-A9D 的反馈时延与信息年龄条款；冻结集中式统一调度与 CTDE 分布式协同调度的并列比较框架，并仅保留文献直接定义的上层任务等待时间。

冲突处理规则：

- 经 `V1.7-A9C` 明确替代的旧条款不再生效。
- 未被明确替代的第一至第八项、任务语义、硬安全约束、上下层接口和事件驱动时间结构继续有效。
- Codex 不得根据代码重构、已有 MAPPO 工程或版本号静默恢复旧的“中央 PPO 最终方法”结论。
- 正式修订至少记录：`amendment_id`、`operation`、`target_clause`、`approved_on`、受影响的公式、接口、测试和文件。

## 2. Codex 强制规则

Codex 必须：

- 修改代码前阅读 `README.md`、`docs/current_task.md`、`docs/model_specification.md` 和 `docs/literature_basis.md`。
- 保持上层任务调度与下层路径执行职责分离；下层不得重新决定全局任务归属。
- 同时维护两套并列的上层实现：
  - `CENTRALIZED_PPO`：单一中央策略基于全局状态生成联合任务分配；
  - `CTDE_MAPPO`：训练时使用集中式 Critic，执行时由平台 Actor 根据规定观测独立选择任务。
- 将研究问题表述为“集中式统一调度与 CTDE 分布式协同调度的适用边界”，不得表述为“PPO 与 MAPPO 谁更先进”。
- 核心架构对比使用相同的完整可行任务集、任务实例、平台参数、下层成本、硬约束、奖励口径、环境交互预算、随机种子和评估场景。
- Centralized PPO 使用硬掩码自回归联合分配；每生成一个平台动作后更新临时任务、平台和回收点占用，禁止同批次重复分配同一非联盟任务。
- CTDE-MAPPO 的同类型平台采用参数共享；UAV 与 USV 可采用类型条件共享网络或两个类型 Actor，不得用 `agent_id` 作为策略特征。
- CTDE-MAPPO 执行时各平台独立提交动作；训练时 Critic 可使用完整全局状态。
- 多个平台同时选择同一非联盟任务时，所有冲突动作均判为无效，任务保持开放，平台等待下一调度事件；必须记录冲突，禁止静默改派或由隐藏中央匹配器选出获胜平台。
- 仅考虑上层任务调度等待时间：任务到达/释放后至获得有效平台分配之间的时间。
- 严格采用文献定义：`WT_j(t) = t - release_time_j`；有效分配时固定为 `d_sch_j = assignment_time_j - release_time_j`。
- 任务未分配到作业窗口结束时，按 `H - release_time_j` 计入最终等待统计，避免通过不分配困难任务降低平均等待时间。
- 等待时间作为次级目标和评价指标；任务完成、质量验收、硬安全约束、截止与重访责任仍为主要目标。
- 保留原有预计旅行/服务时间和实际执行反馈，但不额外建立下层执行时延、通信时延、反馈时延、信息年龄或噪声分布模型。
- 将学习型软锁定 Top-K 降为架构选择之后的扩展机制；核心 Centralized PPO 与 MAPPO 对比不得一方使用学习型 Top-K、另一方使用完整任务集。
- 至少完成两类对比：自然信息架构对比，以及信息条件控制消融。
- 将所有正式变更同步记录到 `docs/experiment_log.md`。

Codex 禁止：

- 在实验完成前指定 Centralized PPO 或 MAPPO 为最终上层方法。
- 将 `MAPPO` 写成完全分布式训练；其正式语义是集中训练、分布执行（CTDE）。
- 将“每个平台独立决策”等同于“每个平台必须使用互不共享的独立网络参数”。
- 为 MAPPO 冲突动作使用策略概率最大者、匹配度最大者或固定平台优先级进行中央仲裁。
- 使用中央学习型 Top-K 只为 Centralized PPO 缩小候选集，再将结果解释为调度架构差异。
- 将任务等待时间定义为任务完成时间、旅行时间、服务时间或反馈返回时间。
- 恢复 `tau_feedback`、`occurred_at/delivered_at`、`information_age` 或“首次服务开始时延”作为当前冻结模型字段。
- 自行构造等待暴露积分、信息时延惩罚或下层随机成本，并声称来自本次指定文献。
- 仅比较最终总回报而不报告完成率、等待时间、冲突率、无效动作率、推理时间和规模变化表现。
- 在未进行平台数量变化测试前声称 MAPPO 必然可扩展；在未进行大规模测试前声称中央 PPO 必然发生动作空间失控。
- 将理论上的单点故障、通信失效或抗毁性写成实验结论，因为当前模型不模拟这些因素。

## 3. 范围状态

当前已冻结：

- 第一至第八项的任务、时间、能量、安全返航、补能、状态转移和上下层接口语义。
- 上层采用事件驱动 SMDP，不采用固定分钟步长。
- 上层研究的核心问题为集中式统一调度与 CTDE 分布式协同调度的架构比较。
- Centralized PPO 与 CTDE-MAPPO 是并列主方案，最终适用边界由实验确定。
- 核心比较使用完整可行任务集；规则 Top-K 可作为双方共享的控制实验；学习型 Top-K 仅作后续扩展。
- 简单时延仅指上层任务等待时间，不含下层执行、通信和反馈时延。
- MAPPO 冲突动作的统一无效处理规则。
- 完成率、逾期率、平均/P95等待时间、冲突率、无效动作率、推理时间和规模测试为核心指标。

当前未冻结：

- 第十项：点、线、面任务的 UAV/USV 下层强化学习执行器及传统规划临时替代方案。
- 两种上层架构的具体网络层数、学习率、熵系数、批量大小等数值超参数。
- 最终选择哪种调度架构，或是否形成分场景适用边界结论。
- 学习型 Top-K 在选定架构上的最终扩展形式。
- 最终实验结果、统计显著性、创新点和论文题目。

## 4. 变更协议

冻结条款仅可在以下条件全部满足时修改：

1. 明确说明冲突、新证据或实现闭合需要。
2. 指定 `AMENDS` 或 `REPLACES`，并准确指出目标条款。
3. 列出受影响的公式、数据结构、接口、测试和文档。
4. 获得用户明确批准。
5. 同一改动中更新 `docs/model_specification.md` 与 `docs/experiment_log.md`。

未经批准的重构可以改善代码质量，但不得改变模型语义。
