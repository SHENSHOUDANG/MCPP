# 当前设置下的近似最优覆盖步数估计

## 设置

- 地图：`20 x 20`
- 障碍率：`0.05`
- 可通行格数量：约 `380`
- agent 数：`4`
- 起点：全部从 depot `(0, 0)` 出发
- 口径：仅计算覆盖完成，不计算返回 depot

代码确认：当 `use_depot=true` 时，所有 agent 的起点均为 depot；当前配置 depot 为 `(0, 0)`。

## 理论下界

初始时 4 个 agent 都在同一个起点 `(0, 0)`，因此初始只覆盖 1 个唯一格子。

需要新增覆盖的格子数：

```text
380 - 1 = 379
```

每个环境 step 中，最多 4 个 agent 各自覆盖 1 个新格子，因此面积下界为：

```text
ceil(379 / 4) = 95 steps
```

最远曼哈顿距离下界为：

```text
distance((0,0), (19,19)) = 38 steps
```

因此主导下界是面积下界：

```text
理论最优下界 ≈ 95 steps
```

## 条带分区近似

使用 4 个纵向条带进行理想蛇形覆盖：

```text
cols 0-4, 5-9, 10-14, 15-19
```

在障碍率 0.05、可通行格约 380 的设置下，按“到达条带入口距离 + 条带内可通行格数量 - 1”估计，20 个随机测试地图和训练 seed 的条带 makespan 均约为：

```text
108 steps
```

无障碍 20x20 地图下，同样 4 条带蛇形覆盖的参考值约为：

```text
114 steps
```

障碍会减少需要覆盖的格子数，但也可能带来绕行；因此工程上更稳妥的近似最优区间是：

```text
约 110-125 steps
```

## 与当前模型对比

如果仅按覆盖完成口径估计，当前设置下的“近似最优”大致可取：

```text
理想下界：95 steps
朴素可行近似：108 steps
实际合理最优区间：110-125 steps
```

此前完整四课程 GAT-CIR 报告的平均环境 steps 为 `174.75`。如果该值按环境 `done` 统计，在当前配置 `require_return_to_depot=true` 时可能包含返回 depot 的尾段；因此若论文严格“不考虑任务返回”，应额外统计 `coverage_completed` 时刻的 step，并用它与上述 `110-125` 的覆盖最优区间比较。
