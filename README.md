# README.md

> 规范版本：V1.7

## 项目名称

**UAV-USV 异构港区巡检集中式与 CTDE 协同调度比较及双层强化学习执行**

研究从交通运输系统、港区运营组织、通航安全和设施运维出发。上层关注任务组织方式与调度效率，下层负责点、线、面任务的旅行和服务路径，不研究通信网络优化或底层控制器设计。

## 核心研究问题

本文不预设“PPO优于MAPPO”或“MAPPO优于PPO”，而是研究：

> 在相同任务环境、约束、奖励和训练预算下，集中式统一调度与集中训练—分布执行的协同调度，哪一种更适合 UAV-USV 港区异构巡检任务分配？二者在任务规模、平台规模、动态任务到达和信息条件变化下的适用边界是什么？

算法仅作为两种调度组织方式的实现工具：

- **集中式统一调度** → Centralized PPO；
- **分布式协同调度** → MAPPO（CTDE）。

## 系统边界

### 上层调度

共同负责：任务分配、排序、等待、返航、补能和事件触发重调度。两种架构使用同一任务语义、完整可行关系和硬安全约束。

#### 架构 A：Centralized PPO

- 一个中央调度策略掌握完整全局状态；
- 通过硬掩码自回归方式生成各平台的联合任务分配；
- 在动作生成过程中更新临时资源占用，因此可在策略内部消除同批任务冲突；
- 代表港区集中控制中心统一组织 UAV 与 USV 的调度方式。

#### 架构 B：CTDE-MAPPO

- 训练阶段使用集中式 Critic 观察全局状态；
- 执行阶段由每个平台 Actor 根据规定观测独立选择任务；
- 同类型平台共享参数，UAV 与 USV 通过类型和能力特征区分；
- 代表平台分布式协同形成任务选择的调度方式。

### 下层执行

最终由强化学习执行器负责 UAV/USV 的旅行路径和点、线、面服务路径。研究初期可用接口兼容的传统规划器临时替代。下层必须向两种上层架构返回完全相同的可行性、时间、能耗、退出位置、完成度、质量验收和失败信息。

## 任务体系与实证边界

一级任务族保持为：

- `HYDROGRAPHIC_SURVEY`；
- `SURFACE_SAFETY_PATROL`；
- `WATERSIDE_ASSET_INSPECTION`。

几何形式保持为 `TARGET | CORRIDOR | AREA`，释放方式保持为 `PERIODIC | SCHEDULED | EVENT`。主要数据驱动实证场景仍为洛杉矶港；旧洋山港数据仅作历史对照。

## 核心公平比较协议

核心比较必须满足：

1. 相同任务实例、任务释放序列和平台配置；
2. 相同完整可行任务集和硬动作掩码；
3. 相同下层成本估计与实际反馈；
4. 相同奖励函数、终局责任和等待时间定义；
5. 相同训练环境交互预算、随机种子和评估场景；
6. 相同模型容量量级，并报告参数量；
7. 不使用只服务于其中一方的学习型 Top-K 候选缩减；
8. MAPPO 冲突不得由隐藏中央优化器消除。

## 文献直接采用的简单等待时间

参考 Zhang 与 Ou（2025）对任务等待时间的定义：

```text
WT_j(t) = t - release_time_j
```

任务获得有效平台分配时：

```text
d_sch_j = assignment_time_j - release_time_j
```

该指标只描述任务在上层待调度池中等待分配的时间，不包括：

- 实际旅行时间；
- 下层服务时间；
- 通信传输时间；
- 执行器反馈时间。

训练和评价使用总等待时间与平均等待时间。任务在作业窗口结束时仍未分配的，按窗口结束时刻计算其等待时间，避免“少分配任务反而平均等待更低”。

## Top-K 与波次机制的新角色

- **核心架构对比**：双方均使用完整可行任务集。
- **控制实验**：可由环境为双方生成完全相同的规则 Top-K。
- **学习型软锁定 Top-K**：不再预设为最终方法，只能在核心架构比较完成后作为扩展研究。

因此，Top-K 不得成为 Centralized PPO 的隐藏优势，也不得掩盖集中式和分布式决策本身的差异。

## MAPPO 冲突处理

若多个平台在同一决策批次选择同一非联盟任务：

```text
conflict_count(j,t) = sum_i I(a_i,t = j)
```

当 `conflict_count(j,t) > 1` 时：

- 所有相关动作均无效；
- 任务保持 `ACTIVE`；
- 相关平台本轮不取得任务；
- 记录冲突和无效动作；
- 下一调度事件重新决策。

不按策略概率、匹配度或固定平台顺序选出获胜者，避免在 MAPPO 后面隐藏一个中央调度器。

## 主要评价指标

- 任务完成率与 MANDATORY 任务完成率；
- 逾期任务率与重访违约；
- 已分配任务平均等待时间；
- 全部释放任务的截尾平均等待时间；
- P95 调度等待时间；
- 冲突批次率、冲突动作率和无效动作率；
- 每次调度平均/P95推理时间；
- 训练样本效率与随机种子稳定性；
- 不同平台数量、任务数量、UAV-USV组合和动态任务到达率下的性能。

## 当前状态

`V1.7-A9C` 已正式替换以下旧结论：

- 不再把双时间尺度中央图 PPO 预设为唯一最终上层方法；
- 不再把 MAPPO 限定为陪衬性基线；
- 删除 V1.6 中的反馈时延、信息年龄和首次服务开始时延；
- 将上层等待时间严格限定为任务释放至有效分配；
- 将学习型 Top-K 降为架构选择后的扩展机制。

当前正式审查任务仍为第十项“下层强化学习路径执行器及传统规划临时替代边界”。下层必须同时服务于 Centralized PPO 与 CTDE-MAPPO，不得改变二者的公平比较条件。
